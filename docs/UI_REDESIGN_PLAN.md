# 午夜音乐记忆展览 UI 重构设计方案

## 1. Summary

将当前 demo 从“音乐评论故事二创工具页”重构为“深夜唱片展览馆”体验。用户进入页面后，先被沉浸式展览空间、3D 唱片封面和精选评论展品吸引，再被自然引导去生成自己的私人唱片展品。

核心目标：

- 第一眼传达“音乐记忆展览”的情绪，而不是普通表单工具。
- 使用生成素材提升空间感、艺术气息和唱片封面质感。
- 保留现有前端模拟生成逻辑，不接真实 AI API。
- 保留现有数据加载、搜索筛选、弹窗选评论、随机逛展、故事/分镜/标签切换等功能。

## 2. Visual Direction

整体方向采用“午夜私人唱片展厅”：

- 色彩：深黑、古铜金、暖琥珀、暗红、低饱和青绿色。
- 氛围：暗场展览馆、玻璃展柜、唱片反光、旧胶片颗粒、暖色聚光。
- 质感：唱片封套、磨砂纸、黑胶纹理、玻璃反射、轻微划痕。
- 文案气质：克制、电影感、情绪召唤，不做营销式大白话。

首屏主文案保持：

> 每一条热评，都是一张没有发行的唱片

辅助文案强调：

> 把陌生人的一句评论，炼成一段会发光的回忆。

## 3. Asset Usage

图片素材只负责艺术质感和情绪氛围，不承担核心交互。所有关键文字、数据、状态和按钮仍由 HTML/CSS/JS 渲染。

- 不把按钮、标题、评论内容做进图片。
- 如果素材中出现不理想文字，只作为被遮罩的背景纹理使用。
- 3D 唱片封面标题、歌曲名、故事摘要由代码动态绘制。

## 4. Layout Redesign

### 4.1 全局结构

页面改为展览馆动线：

1. 左侧展厅导航。
2. 沉浸式 hero 展厅入口。
3. 最新展品墙。
4. 生成私人唱片展品。
5. 私人展品结果区。

桌面端建议布局：

- 左侧固定导航宽度约 `88px` 到 `112px`。
- 主内容区域占满剩余宽度。
- hero 首屏高度接近 `100vh`，下方露出展品墙一点点，提示可继续滚动。

移动端建议布局：

- 左侧导航收为顶部横向栏。
- hero 改为单列：3D 唱片在上，标题和 CTA 在下或叠加在安全区域。
- 展品墙横向滑动或单列卡片，避免横向溢出。

### 4.2 左侧展厅导航

新增固定侧边栏，参考概念图：

- 顶部：品牌标识与 `午夜音乐记忆展览`。
- 中段导航项：`展厅`、`我的展品`、`收藏夹`、`关于展览`。
- 底部：`深夜模式` 开关与年份信息。

交互：

- 点击 `展厅` 滚动到展品墙。
- 点击 `我的展品` 滚动到生成结果或输入区。
- 点击 `收藏夹` 第一版可只做静态入口，不新增收藏逻辑。
- 点击 `关于展览` 可滚动到 footer 或打开轻量说明弹层。

### 4.3 Hero 展厅入口

Hero 使用 `hero-gallery-bg.png` 作为背景：

- 背景左侧加深色渐变，保证标题可读。
- 右侧保留 3D 唱片封面，和背景里的唱片展柜形成层次。
- 标题使用宋体/衬线中文显示，大字号、低行高、展览海报感。
- CTA 使用一个主按钮 `进入展厅`，一个次按钮 `随机逛展`。

Hero 中保留现有数据指标，但视觉弱化为展览馆展签：

- 展出歌曲
- 收录热评
- 歌手
- 万次共鸣

### 4.4 3D 唱片封面

继续使用 Three.js 实现，不改为静态图片。

调整方向：

- 唱片封套更大，视觉中心更靠右。
- 黑胶从封套后方半露出，默认慢速旋转。
- 使用 `cover-midnight-loop.png` 作为初始封面纹理底图。
- 生成故事后，CanvasTexture 继续动态写入标题、二创方向和情绪强度。
- 鼠标移动时产生轻微视差，移动端保持自动呼吸动效。

### 4.5 展品墙

将当前所有展品网格改为更像“展览墙 · 最新展品”的区域：

- 首屏下方优先展示 5 张精选展品卡，对应 5 张生成封面。
- 卡片比例接近唱片封面展品，封面图上叠加半露黑胶。
- 卡片包含：歌曲名、评论摘录、情绪标签、点赞数。
- 保留搜索与歌手筛选，但改为更轻的展墙工具条。
- 点击卡片仍打开当前评论选择弹层。

精选封面映射建议：

- 第 1 张：`exhibit-cover-afterwards.png`
- 第 2 张：`exhibit-cover-rain-night.png`
- 第 3 张：`exhibit-cover-blue-pencil.png`
- 第 4 张：`exhibit-cover-daily-light.png`
- 第 5 张：`exhibit-cover-old-rain.png`

超过 5 张的数据卡可以循环使用这 5 张封面，或继续使用现有 tone class 作为 fallback。

### 4.6 生成私人唱片展品

输入区不再像后台面板，改为“创作装置”：

- 标题：`生成你的私人唱片展品`
- 输入提示：`贴一条让你记很久的评论`
- 主输入框占主要视觉。
- 歌曲名、二创方向、情绪强度作为辅助控制。
- 主按钮：`生成我的展品`

生成后：

- 自动更新 3D 唱片封面。
- 自动切换或滚动到结果区。
- 结果区标题改为 `你的私人唱片展品`。

### 4.7 结果区

结果区从普通 tab 面板改为展品详情：

- 故事：像展览旁白，逐段排版。
- 分镜：像胶片格或展陈说明卡。
- 标签：像展签/情绪印章。
- 仍保留 `故事 / 分镜 / 标签` 三个切换入口。

## 5. Motion Design

动效目标：像“进入展厅、靠近展柜、压制唱片”，避免普通网页弹跳感。

建议实现：

- 页面入场：背景暗光先亮，标题和按钮错峰浮现，3D 唱片随后进入。
- 3D 唱片：慢速旋转、轻微上下呼吸、鼠标视差。
- 展品卡：滚动进入视口时逐张显影。
- 卡片 hover：轻微前移、边框暖光、黑胶慢转、封面轻微放大。
- 生成过程：按钮进入 `generating` 状态，3D 封面扫光，结果区延迟显影。
- 结果显示：故事逐段淡入，分镜逐条出现，标签短促弹出。

技术实现：

- CSS `@keyframes` 负责入场、hover、扫光、显影。
- `IntersectionObserver` 负责滚动进入视口后添加 reveal class。
- Three.js 保留当前 requestAnimationFrame，加入生成状态视觉参数。
- 必须支持 `prefers-reduced-motion: reduce`，关闭非必要动画。

## 6. Implementation Notes

主要修改文件：

- `index.html`
- `styles.css`
- `app.js`

不修改或尽量不动：

- `data/songs.json`
- `data/embedded.js`
- 现有数据抓取脚本

具体实现约束：

- 保留现有关键 DOM id，避免破坏事件绑定。
- 如果重命名 class，确保 JS 查询不受影响。
- 保留 `hero3D.update(title, subtitle)` 的调用方式。
- 搜索、筛选、弹层、生成按钮、tab 切换必须继续工作。
- 不新增真实 AI API，不新增后端。
- 不引入大型动画库；现有 Three.js 继续使用。

建议新增常量：

- `EXHIBIT_COVERS`：5 张展品封面路径数组。
- `DEFAULT_COVER_TEXTURE`：默认唱片封面路径。
- `ASSET_BASE`：`./assets/generated/`。

## 7. Implementation Phases

### Phase 1：接入素材和基础视觉

- 在 CSS 中接入 `hero-gallery-bg.png` 和 `texture-grain-overlay.png`。
- 将默认 3D 封面底图替换为 `cover-midnight-loop.png`。
- 展品卡封面接入 5 张精选素材。
- 保持现有布局基本不大动，先确认素材加载和视觉方向。

### Phase 2：重构页面结构

- 增加左侧展厅导航。
- 重构 hero 为展览入口构图。
- 将展品墙调整为概念图式横向/网格展柜。
- 将输入区改为私人唱片展品生成装置。

### Phase 3：结果区和交互动效

- 结果区改为私人展品详情。
- 加入滚动显影、卡片 hover、生成中状态、结果逐段出现。
- 完成 reduced motion 支持。

### Phase 4：响应式和验收

- 桌面端检查首屏构图。
- 当前 in-app browser 宽度检查无溢出。
- 移动端检查标题、3D 唱片、卡片和输入区可读。
- 完成功能回归和 console 检查。

## 9. Acceptance Criteria

- 首屏第一眼能传达“午夜音乐记忆展览馆”。
- 用户不需要解释，就能理解“评论可以生成唱片展品和二创故事”。
- 生成流程仍然顺畅，且比原来更像一次创作仪式。
- 所有现有核心功能不回退。
- 素材和代码边界清晰，后续可以继续替换素材或接真实 AI。

## 10. Assumptions

- 第一版面向普通听歌用户，不优先展示 B 端创作者能力。
- 当前生成素材已作为 v1 视觉资产使用。
- 真实 AI API 暂不接入，现阶段仍用前端规则模拟。
- 如果后续要上线，需要再做图片压缩、懒加载和版权/来源说明。
